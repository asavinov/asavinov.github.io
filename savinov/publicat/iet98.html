ИНДУКЦИЯ НЕЧЕТКИХ ПРАВИЛ ИЗ ТАБЛИЦ ДАННЫХ


Александр А. Савинов
Институт математики, Академия наук Молдовы

В настоящее время актуальным является направление интеллектуального анализа данных или data mining, главной задачей которого является обнаружение потенциально полезных зависимостей (знаний) в больших массивах данных. В этой области используются такие подходы, как статистика, нейронные сети, генетические и эволюционные алгоритмы, деревья решений и др. Одним из интересных направлений является индукция правил в виде логических зависимостей между признаками.

Пусть исходные данные заданы в виде таблицы, в которой каждая строка (запись) состоит из n значений Будем предполагать, что каждая переменна или признак, соответствующие столбцам таблицы, принимает конечное число значений. В этом случае пространство всевозможных комбинаций значений (универсум) является конечным. Содержимое таблицы можно представить в виде распределения над универсумом, которое каждой точке приписывает число раз (частота), которое соответствующая запись появлялась в таблице.

Для выявления закономерностей, которым подчиняются данные в таким образом представленной таблице был использован подход многомерного нечеткого анализа. На его основе был разработан оригинальный алгоритм индукции нечетких правил. Формальной основой этого алгоритма является процедура преобразования нечеткой ДНФ в нечеткую КНФ, состоящую из всех простых дизъюнкций [1]. Для внутреннего представления данных и знаний используется метод нечетких секционированных векторов. Каждая запись из таблицы данных представляется с помощью одного секционированного вектора, интерпретируемого как конъюнкция, а с помощью набора таких конъюнкций, интерпретируемого как нечеткая ДНФ, можно задать любую таблицу данных (с конечным числом значений в колонках). Двойственным образом, один секционированный вектор, интерпретируемый как дизъюнкция, задает одну логическую зависимость между признаками.

Основной проблемой этого алгоритма является его вычислительная сложность — для реальных задач это преобразование в чистом виде невыполнимо. Поэтому он был модифицирован таким образом, чтобы генерировать наиболее информативные, важные зависимости. Кроме этого, было предложено ряд эвристик, которые позволяют существенно упростить само преобразование, а также специальные индексные структуры для оптимизации доступа к основным структурам данных. Данный алгоритм индукции нечетких правил был реализован в виде нескольких рабочих вариантов, на которых отлаживались различные методы оптимизации. Последняя версия была реализована в виде системы Человек для Windows 95.
